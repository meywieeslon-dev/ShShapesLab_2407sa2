Теоретическое обоснование

Порождающие паттерны проектирования применяются для организации процесса создания объектов таким образом, чтобы клиентский код не зависел от конкретных классов. Они позволяют скрыть детали создания экземпляров и сделать архитектуру более гибкой и удобной для расширения.

Паттерн «Фабричный метод» предусматривает объявление общего интерфейса создания объекта, при этом выбор конкретной реализации передаётся подклассам. Такой подход используется в случаях, когда тип создаваемого объекта может изменяться или заранее неизвестен. Он позволяет вынести логику создания в отдельные классы и тем самым упростить сопровождение и модификацию системы. Паттерн «Абстрактная фабрика» развивает эту идею, объединяя несколько методов создания в единый интерфейс. Он предназначен для формирования группы взаимосвязанных объектов, которые должны использоваться совместно. Благодаря этому клиент работает не с отдельными классами, а с целым семейством согласованных объектов, что упрощает структуру кода и повышает его масштабируемость.

В контексте данной лабораторной работы ожидается, что:

1)Не зависит от конкретных реализаций фигур.

2)Позволяет добавлять новые варианты объектов (например, новый цвет) без изменения клиентского кода.

3)Демонстрирует принципы инкапсуляции, слабой связанности и расширяемости.

4)Соответствует базовым принципам объектно-ориентированного проектирования, в частности принципу открытости/закрытости (OCP).

Описание выполненных действий

Для достижения ожидаемого результата были выполнены следующие действия:

1)Базовая иерархия фигур: В папке /Figures создан абстрактный класс Figure с общим свойством Color и методом для получения UI-элемента. Реализованы конкретные классы Circle, Square, Triangle, наследующие Figure и создающие соответствующие элементы WPF (Ellipse, Rectangle, Polygon). АНАЛОГИЧНЫЕ ДЕЙСТВИЯ ВЫПОЛНЕНЫ В отношении остальных классов фигур.

2)Этап 1: Реализация паттерна «Фабричный метод»: В папке /Creators созданы классы создателей CircleCreator, SquareCreator, TriangleCreator. Каждый создатель переопределяет фабричный метод для инстанцирования конкретного типа фигуры. АНАЛОГИЧНЫЕ ДЕЙСТВИЯ ВЫПОЛНЕНЫ В классах SquareCreator и TriangleCreator. В MainWindow.xaml.cs реализована логика использования создателей для генерации фигур.

3)Этап 2: Рефакторинг до паттерна «Абстрактная фабрика»: В папке /Factories создан интерфейс IFigureFactory с методами CreateCircle(), CreateSquare(), CreateTriangle(). Реализованы конкретные фабрики RedFactory, BlueFactory, GreenFactory, возвращающие наборы фигур соответствующего цвета. АНАЛОГИЧНЫЕ ДЕЙСТВИЯ ВЫПОЛНЕНЫ В классах BlueFactory и GreenFactory. Клиентский код (MainWindow.xaml.cs) переписан для работы исключительно через интерфейс IFigureFactory. При выборе цвета в ComboBox инициализируется соответствующая фабрика, через которую создаются все фигуры.
